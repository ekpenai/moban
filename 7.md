# 微信小程序前端需求说明：模板漏孔 / 蒙版适配

> 适用对象：微信小程序前端同学
>
> 目标：让小程序前端正确渲染模板图片、蒙版漏孔效果，并支持替换图片。

## 1. 接口返回要求

模板接口需要返回完整画布信息：

- `width`：模板画布宽度
- `height`：模板画布高度
- `layers`：按从底到顶排序的图层数组

所有图层的 `x / y / width / height` 都必须基于同一套画布坐标系。

## 2. 图片图层基础字段

每个图片图层至少需要包含：

- `id`
- `type: 'image'`
- `name`（建议可替换图层名称里包含“替换”）
- `url`
- `x`
- `y`
- `width`
- `height`
- `scaleX`
- `scaleY`
- `rotation`

如果后端返回了 `isReplaceable: true`，前端应优先以该字段判断是否允许替换。

## 3. 蒙版字段兼容

前端需要兼容以下蒙版字段任意一种：

- `maskUrl`
- `mask_url`
- `maskSrc`
- `maskPath`
- `mask`
- `mask.url`
- `mask.src`
- `mask.path`
- `mask.image`
- `maskInfo.url`
- `clippingMask.url`
- `clipMask.url`

推荐后端统一返回 `maskUrl`。

## 4. 蒙版效果规则

蒙版图是给图片做“漏孔”效果用的。

约定：

- 白色 / 不透明区域：显示用户图片
- 黑色 / 透明区域：隐藏用户图片
- 蒙版图片最好是 PNG
- 蒙版尺寸尽量与图片图层显示区域一致

## 5. 图层顺序

`layers` 数组必须按视觉顺序返回：

- 越靠前的图层越底层
- 越靠后的图层越上层

如果有前景装饰或带洞遮罩层，必须放在替换图片图层之后。

## 6. 替换图片交互

图片图层双击或点击替换按钮时：

1. 判断该层是否可替换
2. 选择本地图片
3. 按图层或蒙版区域的比例进行裁剪
4. 上传到后端
5. 更新该图层的 `url`
6. 保留原始蒙版信息，避免漏孔效果丢失

## 7. 小程序页面使用建议

前端页面至少需要支持：

- 模板列表加载
- 模板详情加载
- 图片替换
- 蒙版漏孔渲染
- 导出后轮询任务状态
- 最终展示导出图片

## 8. 接口关注点

请重点确认以下接口返回正确：

- `GET /templates`
- `GET /templates/:id`
- `POST /templates/save`
- `POST /render`
- `GET /render/:jobId`

## 9. 最后确认清单

上线前请确认：

- 图片 URL 可被微信小程序直接访问
- 蒙版 URL 可直接访问且是 HTTPS
- 合法域名已配置
- 图层顺序正确
- 可替换图片层返回 `isReplaceable` 或名称包含“替换”
