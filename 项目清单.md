# 项目清单（精简执行版）

> 目标：把本项目部署到 Sealos，让微信小程序调用 API，完成「获取模板 -> 编辑 -> 保存/导出」。

---

## A. 今日必须完成（按顺序）

- [ ] 1. 在 Sealos 创建 `MySQL` 和 `Redis`
- [ ] 2. 部署 `moban-server`（使用 `server/Dockerfile`）
- [ ] 3. 部署 `moban-worker`（使用 `worker/Dockerfile`）
- [ ] 4. 给 `moban-server` 绑定 HTTPS 域名
- [ ] 5. 小程序后台配置 `request 合法域名`
- [ ] 6. 真机发起一次导出并成功拿到图片

- [ ] `REDIS_PORT`（通常 6379）

## 服务
- [ ] `PORT=3000`
- [ ] `NODE_ENV=production`

---

## C. Sealos 部署配置（直接照填）

## `moban-server`
- 工作目录：`server`
- Dockerfile：`server/Dockerfile`
- 端口：`3000`
- 启动：`npm run start:prod`
- 环境变量：填 B 节所有 DB/Redis + `UPLOAD_LIMIT=50mb` + `CLEANUP_DAYS_LIMIT=7`

## `moban-worker`
- 工作目录：`worker`
- Dockerfile：`worker/Dockerfile`
- 不暴露端口
- 启动：`npx ts-node index.ts`
- 环境变量：填 B 节所有 DB/Redis + `NODE_ENV=production`

---

## D. 小程序接入（必须）

- [ ] 小程序请求基地址改为：`https://你的API域名`
- [ ] 微信公众平台配置合法域名（HTTPS）
- [ ] 小程序调用接口：
  - `GET /templates`
  - `GET /templates/:id`
  - `POST /templates/save`

---

## F. 常见问题（快速定位）

## 1) 一直“正在导出”
- worker 没启动，或 Redis 配置错误。

## 2) 导出失败：`Image to composite must have same dimensions or smaller`
- 图层越界导致；当前 worker 已加入裁剪逻辑，重启 worker 生效。

## 3) 小程序请求失败
- 域名不是 HTTPS，或未在小程序后台加入合法域名。

## G. 本地演练命令（可选）

已准备文件：
- `server/Dockerfile`
- `worker/Dockerfile`
- `docker-compose.sealos.local.yml`

启动：

```bash
docker compose -f docker-compose.sealos.local.yml up -d --build
```

停止：

```bash
docker compose -f docker-compose.sealos.local.yml down
```
