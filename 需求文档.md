你是一个资深全栈架构师，请帮我从0生成一个完整可运行的Web应用项目。

项目名称：在线模板设计工具（类似Canva的PSD模板编辑器）

====================
一、技术栈要求
====================

前端：
- React + TypeScript
- Konva.js（核心画布编辑）
- Zustand（状态管理）
- TailwindCSS（UI）
- Vite（构建工具）

后端：
- Node.js + NestJS
- MySQL（TypeORM）
- Redis + BullMQ（任务队列）
- Sharp（图片生成）
- ag-psd（PSD解析）
- multer（文件上传）

====================
二、核心功能
====================

1. PSD模板上传
- 上传PSD文件
- 解析图层
- 转换为JSON模板

2. 模板管理
- 获取模板列表
- 获取模板详情

3. 编辑器（核心）
- 渲染JSON模板
- 支持拖拽、缩放、旋转
- 支持文字编辑（双击修改）
- 支持图片替换
- 支持图层选中（Transformer）

4. 图片生成
- 提交编辑后的JSON
- 后端生成图片（Sharp）
- 返回图片URL

5. 渲染队列
- 使用BullMQ
- 支持任务状态查询

====================
三、数据结构（必须使用）
====================

模板JSON格式：

{
  id: string,
  width: number,
  height: number,
  layers: [
    {
      id: string,
      type: "text" | "image",
      x: number,
      y: number,
      width: number,
      height: number,
      rotation: number,
      scaleX: number,
      scaleY: number,
      text?: string,
      fontSize?: number,
      color?: string,
      url?: string,
      editable: boolean
    }
  ]
}

====================
四、API设计
====================

POST /upload/psd
GET /templates
GET /templates/:id
POST /render
GET /render/:jobId

====================
五、项目结构要求
====================

生成完整项目结构：

/project-root
  /web        React前端
  /server     NestJS后端
  /worker     渲染服务

每个模块必须：
- 结构清晰
- 可扩展
- 包含必要代码

====================
六、前端要求
====================

实现一个基础编辑器页面：

- 使用Konva渲染图层
- 支持拖拽（draggable）
- 支持选中（Transformer）
- 双击修改文字
- 支持加载模板JSON

====================
七、后端要求
====================

- PSD解析服务
- 模板存储（MySQL）
- 渲染队列（BullMQ）
- Worker生成图片（Sharp）

====================
八、代码要求
====================

- 所有代码必须完整可运行
- 不要只给伪代码
- 提供关键文件代码
- 提供启动步骤

====================
