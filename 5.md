# 后端模板接口补充要求：蒙版漏孔效果

当前前端已经支持读取图片图层的蒙版字段并按蒙版显示漏孔效果。如果页面仍然没有效果，通常是因为后端返回的模板图层信息不完整，尤其是缺少蒙版图片地址或图层结构没有表达“哪张图片使用哪个蒙版”。

## 1. 接口需要返回完整模板尺寸

模板根对象需要包含画布原始尺寸：

```json
{
  "width": 675,
  "height": 1200,
  "layers": []
}
```

要求：

- `width`：模板设计稿宽度，单位 px。
- `height`：模板设计稿高度，单位 px。
- 所有图层的 `x`、`y`、`width`、`height` 都必须基于这个同一套坐标系。

## 2. 图片图层必须返回基础字段

每个图片图层至少需要返回：

```json
{
  "id": "image_1",
  "type": "image",
  "name": "替换图片1",
  "url": "https://example.com/template/default-photo.png",
  "x": 100,
  "y": 200,
  "width": 300,
  "height": 400,
  "scaleX": 1,
  "scaleY": 1,
  "rotation": 0
}
```

要求：

- `type` 必须是 `image`。
- `url` 必须是可访问的图片地址，微信小程序能直接加载。
- `x`、`y` 是图片图层左上角坐标。
- `width`、`height` 是图片图层原始显示区域大小。
- `scaleX`、`scaleY` 没有缩放时传 `1`。
- `rotation` 没有旋转时传 `0`。
- 如果该图层允许用户替换图片，`name` / `title` / `label` 中需要包含 `替换` 两个字，例如 `替换图片1`。

## 3. 需要漏孔效果的图片图层必须返回蒙版地址

前端目前支持以下任意一种字段格式，后端选择一种统一返回即可。

推荐格式：

```json
{
  "id": "image_1",
  "type": "image",
  "name": "替换图片1",
  "url": "https://example.com/template/default-photo.png",
  "maskUrl": "https://example.com/template/masks/image_1_mask.png",
  "x": 100,
  "y": 200,
  "width": 300,
  "height": 400,
  "scaleX": 1,
  "scaleY": 1,
  "rotation": 0
}
```

也支持对象格式：

```json
{
  "id": "image_1",
  "type": "image",
  "name": "替换图片1",
  "url": "https://example.com/template/default-photo.png",
  "mask": {
    "url": "https://example.com/template/masks/image_1_mask.png"
  },
  "x": 100,
  "y": 200,
  "width": 300,
  "height": 400
}
```

前端可识别的蒙版字段包括：

- `maskUrl`
- `mask_url`
- `maskSrc`
- `maskPath`
- `mask`
- `mask.url`
- `mask.src`
- `mask.path`
- `mask.image`
- `maskInfo.url`
- `clippingMask.url`
- `clipMask.url`

建议后端统一使用 `maskUrl`，最简单清晰。

## 4. 蒙版图片规范

蒙版图片必须是单独的图片资源，例如 PNG。

要求：

- 蒙版图片尺寸建议与图片图层显示区域一致，例如图片图层是 `300 x 400`，蒙版图也最好是 `300 x 400`。
- 白色或不透明区域表示显示用户图片。
- 黑色或透明区域表示隐藏用户图片，也就是形成漏孔。
- 推荐使用带透明通道的 PNG：透明区域隐藏，不透明区域显示。
- 蒙版图片地址必须是 HTTPS，并且在微信小程序后台配置了合法 downloadFile / request 域名。
- 不能只返回 PSD 里的蒙版元数据，前端需要的是小程序可以直接加载的图片 URL。

## 5. 图层顺序必须按设计稿从底到顶返回

`layers` 数组需要按视觉层级排序：

```json
{
  "layers": [
    { "type": "image", "name": "背景图", "url": "..." },
    { "type": "image", "name": "替换图片1", "url": "...", "maskUrl": "..." },
    { "type": "image", "name": "前景装饰", "url": "..." },
    { "type": "text", "name": "标题文字", "text": "..." }
  ]
}
```

要求：

- 越靠前的图层越先绘制，视觉上越靠底层。
- 越靠后的图层越后绘制，视觉上越靠上层。
- 如果漏孔效果依赖前景装饰图，前景装饰图必须作为单独图层放在替换图片图层之后。

## 6. 替换图片图层和前景漏孔图层的关系

如果设计稿中是这种结构：

- 背景图
- 用户可替换图片
- 带洞的前景装饰图

后端应该返回三层：

```json
{
  "width": 675,
  "height": 1200,
  "layers": [
    {
      "id": "bg",
      "type": "image",
      "name": "背景",
      "url": "https://example.com/bg.png",
      "x": 0,
      "y": 0,
      "width": 675,
      "height": 1200
    },
    {
      "id": "replace_photo_1",
      "type": "image",
      "name": "替换图片1",
      "url": "https://example.com/default-photo.png",
      "maskUrl": "https://example.com/mask-photo-1.png",
      "x": 120,
      "y": 260,
      "width": 420,
      "height": 520,
      "scaleX": 1,
      "scaleY": 1,
      "rotation": 0
    },
    {
      "id": "front_decoration",
      "type": "image",
      "name": "前景装饰",
      "url": "https://example.com/front-decoration.png",
      "x": 0,
      "y": 0,
      "width": 675,
      "height": 1200
    }
  ]
}
```

说明：

- `replace_photo_1` 是用户替换图片的位置。
- `maskUrl` 控制用户图片显示成什么形状。
- `front_decoration` 是盖在上面的装饰层，如果有洞，需要前景 PNG 本身带透明通道。

## 7. 如果后端是从 PSD / 设计稿解析模板

请后端在解析时额外导出：

1. 每个普通图片图层的 PNG 地址。
2. 每个可替换图片图层的占位图地址。
3. 每个可替换图片图层对应的蒙版 PNG 地址。
4. 每个图层的 `x`、`y`、`width`、`height`、`rotation`、`scaleX`、`scaleY`。
5. 图层顺序。
6. 图层名称，尤其是可替换图层名称需要包含 `替换`，或额外返回 `isReplaceable: true`。

## 8. 建议增加 isReplaceable 字段

目前前端通过图层名称是否包含 `替换` 判断是否允许用户替换图片。为了更稳定，建议后端直接返回：

```json
{
  "type": "image",
  "name": "用户照片1",
  "isReplaceable": true
}
```

如果后端增加该字段，前端也可以改成优先使用 `isReplaceable`，避免依赖中文图层名称。

## 9. 自查清单

如果前端没有漏孔效果，请后端优先检查：

- 图片图层有没有返回 `maskUrl`。
- `maskUrl` 是否能在浏览器直接打开。
- `maskUrl` 是否是 HTTPS。
- 蒙版图片是否真的有透明/黑白区域，而不是一整张纯白图。
- 蒙版图片尺寸是否和图片图层区域匹配。
- `layers` 顺序是否正确。
- 前景装饰图是否作为独立图层返回。
- 前景装饰图需要透明洞时，PNG 本身是否带透明通道。
- 微信小程序后台是否配置了图片域名。
