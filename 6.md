# PSD 解析后“替换图层”双资源输出需求说明（最终版）

## 1. 需求背景

当前小程序存在两个使用模板图片资源的页面：

- `@pages/template-detail`
  - 用于展示模板详情及普通效果图。
  - 该页面继续使用模板的普通展示资源。

- `@pages/image-editor`
  - 用于替换图片的位置调整、缩放、旋转及蒙版对齐预览。
  - 该页面需要一份用于蒙版展示的加工图资源。

为降低前端对 PSD 透明通道的依赖，建议在后端解析 PSD 源文件时，针对所有“替换”图层同时生成两份资源，并保存到对象存储。

---

## 2. 需求目标

针对 PSD 中标记为“替换”的图层，后端需要同时产出两类资源：

### 2.1 普通展示资源

- 用途：`@pages/template-detail`
- 资源内容：保持图层原始视觉效果
- 说明：该资源当前已有实现，可继续沿用现有存储逻辑

### 2.2 黑白蒙版资源

- 用途：`@pages/image-editor`
- 资源内容：由图层 alpha 通道加工生成的黑白图
- 生成规则：
  - 有像素区域输出白色 `#FFFFFF`
  - 无像素区域输出黑色 `#000000`
- 页面用途：
  - 白色区域对应可见区域，前端可露出用户上传图片
  - 黑色区域对应遮罩区域，可用于灰色背景/非可见区域表现

---

## 3. 资源生成规则

### 3.1 普通资源

对 PSD 原始“替换”图层继续生成并保存普通展示图。

示例：

```json
{
  "type": "image",
  "name": "替换图片1",
  "url": "https://cdn.example.com/template/replace_1.png"
}
```

### 3.2 黑白蒙版资源

在保存普通资源的基础上，再为同一个“替换”图层生成黑白蒙版图。

生成要求：

1. 读取图层 alpha 通道。
2. 按像素判断透明度：
   - `alpha > 0`：白色
   - `alpha = 0`：黑色
3. 输出图必须与原图层尺寸一致。
4. 建议输出格式为 PNG。

示例：

```json
{
  "type": "image",
  "name": "替换图片1",
  "maskUrl": "https://cdn.example.com/template/replace_1_mask_bw.png"
}
```

---

## 4. 接口返回要求

后端在模板接口中，建议对每一个可替换图层返回如下字段：

```json
{
  "id": "replace_1",
  "type": "image",
  "name": "替换图片1",
  "isReplaceable": true,
  "url": "https://cdn.example.com/template/replace_1.png",
  "maskUrl": "https://cdn.example.com/template/replace_1_mask_bw.png",
  "x": 120,
  "y": 260,
  "width": 420,
  "height": 520,
  "scaleX": 1,
  "scaleY": 1,
  "rotation": 0
}
```

### 字段说明

- `url`
  - 普通展示资源地址
  - 用于模板详情页及普通预览

- `maskUrl`
  - 黑白蒙版资源地址
  - 用于图片编辑页进行蒙版对齐预览

- `isReplaceable`
  - 明确标识该图层是否为可替换图层
  - 建议后端直接返回，避免前端依赖图层名称判断

- `x / y / width / height / rotation / scaleX / scaleY`
  - 图层定位与变换信息
  - 用于前端准确渲染和编辑

---

## 5. 对象存储建议

建议将两类资源分开存储，便于排查和维护：

### 5.1 普通资源路径

示例：

- `template/123/original/replace_1.png`
- 或使用当前已有的普通资源目录结构

### 5.2 蒙版资源路径

示例：

- `template/123/mask/replace_1_mask_bw.png`

建议按模板 ID 或图层 ID 进行目录划分，保证资源命名清晰、可追踪。

---

## 6. 页面使用约定

### 6.1 `@pages/template-detail`

- 使用普通展示资源 `url`
- 不依赖 `maskUrl`
- 保持当前展示逻辑不变

### 6.2 `@pages/image-editor`

- 优先使用 `maskUrl`
- 蒙版图用于展示编辑区域边界、辅助用户对齐
- 白色区域对应可见窗口
- 黑色区域对应遮罩区域

### 6.3 页面跳转参数建议

从编辑页进入图片调整页时，建议同步传递：

- `src`：当前用户选择的新图片临时路径
- `layerIndex`：当前图层索引
- `targetWidth` / `targetHeight`：目标图层尺寸
- `templateId`：模板 ID
- `maskPreviewSrc`：该替换图层对应的 `maskUrl`

这样 `@pages/image-editor` 可以直接展示该替换图层的蒙版预览。

---

## 7. 解析 PSD 时的实现建议

请后端在 PSD 解析环节额外完成以下工作：

1. 识别所有“替换”图层。
2. 为每个替换图层导出普通 PNG 资源。
3. 基于图层 alpha 通道生成黑白蒙版图。
4. 将两类资源分别上传至对象存储。
5. 在模板接口中同时返回 `url` 与 `maskUrl`。
6. 同步返回 `isReplaceable` 及完整图层位置信息。

---

## 8. 验收标准

满足以下条件则视为开发完成：

1. 后端模板接口能返回可替换图层的普通资源地址和蒙版地址。
2. 普通资源在 `@pages/template-detail` 页面显示正常。
3. `maskUrl` 在 `@pages/image-editor` 页面可正常加载。
4. 黑白蒙版图的白色区域能准确对应可见区域。
5. 黑白蒙版图的尺寸与原图层尺寸一致。
6. 前端无需再解析 PSD 的透明通道。

---

## 10. 结论

该方案可行，且建议作为当前模板系统的标准输出方式。

通过“普通资源 + 黑白蒙版资源”双输出机制，可以将模板展示与蒙版编辑完全拆分：

- `template-detail` 只负责普通展示
- `image-editor` 只负责基于蒙版的交互编辑

这样可以减少前端逻辑复杂度，提升模板渲染和编辑的一致性与稳定性。
