# 后端需求说明：`@pages/template-editor` 的模板渲染与蒙版合成能力

## 1. 背景

当前小程序的 `@pages/template-editor` 页面承担模板编辑、图层渲染、图片替换、导出成图等核心能力。该页面已经能完成基础编辑交互，但如果要稳定实现“替换图层的漏孔效果”和最终导出图片，建议后端补充以下能力：

- 模板接口返回完整图层数据
- 替换图层返回对应的蒙版资源 `maskUrl`
- 导出时由后端完成最终合成，减少小程序端 canvas 兼容风险

前端当前职责主要是：

- 拉取模板详情
- 识别可替换图层
- 选图、拖拽、缩放、旋转、翻转等交互
- 将渲染参数提交给后端
- 轮询导出任务结果

---

## 2. 目标

请后端提供一套适配 `@pages/template-editor` 的模板数据与导出能力，确保：

1. 模板详情能返回完整画布和图层信息。
2. 可替换图层能返回普通展示资源和黑白蒙版资源。
3. 前端替换图片后，可以将参数提交给后端进行最终合成。
4. 后端返回最终成图 URL，前端只负责展示和保存。
5. 小程序端不需要处理复杂的像素级蒙版合成。

---

## 3. 模板详情接口要求

### 3.1 模板基础信息

建议 `GET /templates/:id` 返回：

```json
{
  "id": "template_123",
  "width": 675,
  "height": 1200,
  "layers": []
}
```

字段说明：

- `width`：模板画布宽度
- `height`：模板画布高度
- `layers`：按视觉顺序排列的图层数组，必须从底到顶返回

所有图层的 `x / y / width / height` 都必须基于同一套画布坐标系。

---

### 3.2 图片图层基础字段

每个图片图层至少需要包含：

- `id`
- `type: 'image'`
- `name`
- `url`
- `x`
- `y`
- `width`
- `height`
- `scaleX`
- `scaleY`
- `rotation`

如果该图层允许用户替换，建议同时返回：

- `isReplaceable: true`

前端目前会优先读取 `isReplaceable`，避免依赖图层名称中是否包含“替换”。

---

### 3.3 蒙版字段要求

对于需要漏孔效果的替换图层，后端需要额外返回蒙版资源地址，推荐字段名：

- `maskUrl`

前端也兼容以下写法，但建议后端统一返回 `maskUrl`：

- `mask_url`
- `maskSrc`
- `maskPath`
- `mask`
- `mask.url`
- `mask.src`
- `mask.path`
- `mask.image`
- `maskInfo.url`
- `clippingMask.url`
- `clipMask.url`

示例：

```json
{
  "id": "replace_1",
  "type": "image",
  "name": "替换图片1",
  "isReplaceable": true,
  "url": "https://cdn.example.com/template/replace_1.png",
  "maskUrl": "https://cdn.example.com/template/replace_1_mask_bw.png",
  "x": 120,
  "y": 260,
  "width": 420,
  "height": 520,
  "scaleX": 1,
  "scaleY": 1,
  "rotation": 0
}
```

---

## 4. 蒙版资源规范

蒙版资源建议由后端在解析 PSD 时生成，规则如下：

- 有像素的地方输出白色 `#FFFFFF`
- 没像素的地方输出黑色 `#000000`
- 输出格式建议为 PNG
- 尺寸必须与原替换图层一致
- 资源地址必须可被小程序直接访问，且为 HTTPS

这样前端在 `@pages/template-editor` 中就能同时完成：

- 替换图层识别
- 蒙版预览
- 替换图片后导出一致性

---

## 5. 导出接口要求

### 5.1 创建导出任务

建议接口：

- `POST /render`

用途：提交模板最终导出任务。

#### 请求参数建议

```json
{
  "templateId": "template_123",
  "width": 675,
  "height": 1200,
  "layers": [
    {
      "id": "replace_1",
      "type": "image",
      "url": "https://cdn.example.com/upload/user-image.png",
      "maskUrl": "https://cdn.example.com/template/replace_1_mask_bw.png",
      "x": 120,
      "y": 260,
      "width": 420,
      "height": 520,
      "scaleX": 1,
      "scaleY": 1,
      "rotation": 0,
      "isReplaceable": true
    }
  ]
}
```

#### 字段说明

- `templateId`：模板 ID
- `width / height`：画布尺寸
- `layers`：当前最终参与导出的图层数组
- `url`：当前图层使用的图片地址
- `maskUrl`：该图层对应的黑白蒙版地址
- `x / y / width / height / scaleX / scaleY / rotation`：用于后端精确重建图层位置和变换

---

### 5.2 查询导出任务状态

建议接口：

- `GET /render/:jobId`

用途：轮询导出任务状态，并在完成后返回最终成图。

#### 返回示例

处理中：

```json
{
  "jobId": "job_123",
  "status": "processing",
  "progress": 60,
  "imageUrl": ""
}
```

成功：

```json
{
  "jobId": "job_123",
  "status": "success",
  "progress": 100,
  "imageUrl": "https://cdn.example.com/render/result_123.png"
}
```

失败：

```json
{
  "jobId": "job_123",
  "status": "failed",
  "error": "mask composition failed"
}
```

---

## 6. 合成规则

后端在生成最终图片时，请按以下规则处理：

1. 读取模板图层顺序。
2. 读取替换图层的当前 `url`。
3. 读取替换图层的 `maskUrl`。
4. 按提交的坐标和变换参数重建图层位置。
5. 使用蒙版进行合成：
   - 白色区域显示用户图片
   - 黑色区域保持遮罩/透明
6. 输出最终图片到对象存储，并返回 `imageUrl`。

---

## 7. 前端配合方式

`@pages/template-editor` 前端会负责：

- 拉取模板详情
- 识别可替换图层
- 点击“替换图片”后传入新图片地址
- 将当前图层的 `maskUrl` 和位置信息一并提交给后端
- 轮询导出任务，拿到最终图片后保存到相册

前端不再要求自己做最终像素级合成，只保留交互和任务提交。

---

## 8. 验收标准

满足以下条件即视为功能完成：

1. `GET /templates/:id` 能返回完整图层数据。
2. 可替换图层能返回 `isReplaceable` 和 `maskUrl`。
3. `POST /render` 能接收模板图层参数并创建导出任务。
4. `GET /render/:jobId` 能正确返回任务状态和最终成图 URL。
5. 小程序端无需本地做复杂像素级蒙版处理。
6. 最终导出的图片与模板蒙版效果一致。

---

## 10. 影响还原度的关键点

如果想让前端效果尽量接近 PSD，以下内容尤为重要：

1. **字体一致**
   - PSD 里用什么字体，后端尽量明确返回并保证前端可用同款字体。

2. **文字排版参数完整**
   - 字号、行高、对齐方式、字重、字间距越完整，排版越接近。

3. **蒙版图必须严格按透明通道生成**
   - 白色区域显示用户图片
   - 黑色区域隐藏用户图片
   - 蒙版尺寸要与图层一致

4. **前景透明洞图要单独导出**
   - 不能只给一张背景图
   - 需要把带洞的前景装饰层拆出来

5. **图片裁切方式要统一**
   - 替换图层的显示区域、缩放比例、旋转角度都要和 PSD 保持一致

6. **尽量让后端输出最终成图**
   - 如果目标是接近 PSD，最终合成交给后端会比小程序本地 canvas 更稳定。

---

## 11. 结论

`@pages/template-editor` 的关键在于：

- 模板详情要完整
- 替换图层要明确
- 蒙版资源要稳定
- 文字和图片的基础排版字段要齐全
- 导出结果尽量交给后端

这样小程序前端就能保持轻量，渲染效果也更稳定，也更接近 PSD 的真实视觉效果。
