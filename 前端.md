# 婚礼请帖小程序前端对接文档（字段动态解析与海报生成）

前端同学们好！本次针对婚礼请帖场景，后端已实现了**动态字段解析**与**防溢出渲染（autoScale）**的全面升级。你不需要在前端写任何繁杂的正则解析，所有的逻辑都已经封装到了全新的后端接口中。

请按照以下流程对接。

---

## 1. 核心流程总览

1. **获取模板详情（包含自动提取的动态字段）**：`GET /templates/:id`
2. **渲染动态表单**：根据接口返回的 `fields` 数组，循环渲染对应的 `<input>`。
3. **收集用户输入**：收集表单数据为一个 `key-value` 对象（如 `{ groom: "李明轩" }`）。
4. **回填字段获取新模板**：调用 `POST /templates/fill` 将数据传给后端，获取到带有用户最新数据的模板对象。
5. **提交渲染**：拿着新的模板对象调用 `POST /render`，然后轮询拿到最终的海报图片。

---

## 2. API 接口文档与对接方式

### 第一步：获取编辑模板与动态字段

- **URL**：`GET /templates/:id`
- **说明**：在这个版本中，返回的数据里多了一个非常关键的 `fields` 字段。后端已经帮你把模板里所有标记了如 `【groom:text|李明轩】` 的图层解析成了结构化的表单配置。

**后端响应示例：**
```json
{
  "data": {
    "id": "tpl_001",
    "name": "婚礼海报",
    "width": 675,
    "height": 1200,
    "layers": [ /* 图层数据 */ ],
    "fields": [
      {
        "key": "groom",
        "label": "新郎",         // 从“新郎:【groom:text】”自动提取出
        "type": "text",          // 类型：text/tel/date
        "value": "李明轩",       // 默认值
        "max": 10,               // 若标记了 |max=10
        "autoScale": true        // 若标记了 |autoScale
      },
      {
        "key": "phone",
        "label": "电话",
        "type": "tel",
        "value": "13800000000"
      }
    ]
  }
}
```

### 第二步：根据 fields 渲染表单（WXML示例）

你只需要遍历 `fields` 即可，不用管图层的事。

```html
<!-- pages/editor/editor.wxml -->
<view class="form-container">
  <view wx:for="{{templateData.fields}}" wx:key="key" class="form-item">
    <view class="label">{{item.label}}</view>
    <!-- 根据 type 渲染不同类型的输入框 -->
    <input
      value="{{item.value}}"
      data-key="{{item.key}}"
      bindinput="onInput"
      type="{{item.type == 'tel' ? 'number' : 'text'}}"
      maxlength="{{item.max || -1}}" 
      placeholder="请输入{{item.label}}"
    />
  </view>
</view>
```

```javascript
// pages/editor/editor.js
Page({
  data: {
    templateData: null, // 从 GET /templates/:id 获取的整体数据
    formData: {}        // 用于收集用户最新输入
  },

  onLoad(options) {
    this.fetchTemplate(options.id);
  },

  async fetchTemplate(id) {
    const res = await wx.request({ url: `https://你的域名/templates/${id}` });
    const data = res.data.data;
    
    // 初始化默认的 formData
    const initData = {};
    if (data.fields) {
      data.fields.forEach(f => {
        initData[f.key] = f.value;
      });
    }

    this.setData({ 
      templateData: data,
      formData: initData
    });
  },

  onInput(e) {
    const key = e.currentTarget.dataset.key;
    const value = e.detail.value;
    this.setData({
      [`formData.${key}`]: value
    });
  }
});
```

### 第三步：合并数据与生成海报（一键搞定）

当用户点击“生成海报”或“保存”时，不要在前端去用正则替换图层的 text。请直接调用新的 `POST /templates/fill` 接口。

- **URL**：`POST /templates/fill`
- **用途**：将用户输入的 `formData` 和原始模板发给后端，后端会自动处理：替换变量、自动截断超出 `max` 长度的文字、打上防溢出标记等。

```javascript
  // 提交并生成海报
  async onGenerate() {
    wx.showLoading({ title: '正在生成中...' });

    try {
      // 1. 调用 fill 接口回填数据
      const fillRes = await wx.request({
        url: 'https://你的域名/templates/fill',
        method: 'POST',
        data: {
          template: this.data.templateData,  // 原始模板数据（包含 layers）
          fieldsData: this.data.formData     // 用户的最新输入 { groom: "张三", phone: "123" }
        }
      });
      
      const newTemplate = fillRes.data.data; // 已经替换好最新文字的模板对象
      
      // 2. 将回填好的模板发去渲染队列
      const renderRes = await wx.request({
        url: 'https://你的域名/render',
        method: 'POST',
        data: { template: newTemplate }
      });
      
      const jobId = renderRes.data.jobId;
      console.log("拿到 jobId 开始轮询：", jobId);

      // 3. 开始轮询导出状态
      this.pollStatus(jobId);

    } catch (err) {
      wx.showToast({ title: '生成失败', icon: 'none' });
    }
  },

  // 轮询逻辑
  pollStatus(jobId) {
    const timer = setInterval(async () => {
      const res = await wx.request({
        url: `https://你的域名/render/${jobId}`,
        method: 'GET'
      });

      if (res.data.status === 'completed') {
        clearInterval(timer);
        wx.hideLoading();
        // res.data.result 是生成的 base64 或图片 url
        console.log("生成成功，图片地址:", res.data.result);
        
        // （你可以将 base64 转成本地文件再保存至相册，此处省略）
      } else if (res.data.status === 'failed') {
        clearInterval(timer);
        wx.hideLoading();
        wx.showToast({ title: '渲染失败: ' + res.data.failedReason, icon: 'none' });
      }
    }, 2000); // 建议 2 秒查一次
  }
```

## 3. 用户换图怎么处理？（关于图片层替换）

如果你还想让用户上传照片（如：替换模板上的婚纱照），流程也很简单：

1. `GET /templates/:id` 后，你不仅可以渲染 `fields` 表单，也可以遍历 `layers`。
2. 找出 `type === 'image'` 且 `editable === true` 的图层。
3. 让用户选择本地相册图片，调用 `POST /upload/image`。
4. 拿到返回的 `url`。
5. 将这个新 `url` 直接赋值给你的 `templateData.layers[x].url`。
6. 然后在最后一步点击生成时，带着这个已经被修改过 `url` 的 `templateData` 去调用 `POST /templates/fill` 和 `POST /render` 即可。

---

## 4. 为什么要用这种方案？

- **安全**：后端包含了文字截断逻辑（`max=10`），防止撑爆海报。
- **排版完美**：如果你在模板设计时加了 `autoScale`（自动缩放）标记，不管文字输入多长，渲染引擎都会将这行文字**挤压**在既定的图层宽度内，绝对不会溢出画布。
- **免维护**：以后只要是在 PSD 里用 `【key:type|value】` 格式新增的字段，小程序无需改一行代码，表单会自动渲染出来！

## 5. 获取指定替换图层的位置和蒙版信息

如果你的应用需要用户在特定图层（如用户照片的**替换区域**）进行裁剪和交互操作，你可以调用专门的 API 来获取名称中包含“替换”的图层信息。

- **URL**：`GET /templates/:id/replace-layer`
- **用途**：返回模板中名称为“替换”的图层信息（坐标 `x`、`y`，尺寸 `width`、`height`，以及默认图片 `url` 等），用于前端图像裁剪组件或蒙版定位的初始化。

**后端响应示例：**
```json
{
  "data": {
    "id": "12345678-abcd-1234-abcd-1234567890ab",
    "name": "替换",
    "x": 100,
    "y": 200,
    "width": 300,
    "height": 400,
    "url": "https://你的域名/uploads/xxx.png",
    "type": "image"
  }
}
```

**前端示例逻辑：**
```javascript
// 获取替换图层信息，准备打开裁剪蒙版
async fetchReplaceLayer(templateId) {
  const res = await wx.request({ 
    url: `https://你的域名/templates/${templateId}/replace-layer` 
  });
  
  if (res.data.data) {
    const layerInfo = res.data.data;
    // 使用这个 layerInfo 的 x, y, width, height 去初始化你的裁剪组件或蒙版
    this.setData({
      maskRect: {
        x: layerInfo.x,
        y: layerInfo.y,
        width: layerInfo.width,
        height: layerInfo.height
      }
    });
  } else {
    console.log(res.data.message); // 未找到名称为“替换”的图层
  }
}
```
